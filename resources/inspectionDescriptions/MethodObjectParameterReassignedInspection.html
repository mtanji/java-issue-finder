Here, the code is reassigning the method Object parameters and this is considered a bad practice.
Allowing this technique reduces the code readability as other developers will not be able to know whether
the original parameter reference is used or a reassigned reference without scanning the whole method.
In other words, it can be confusing for other developers that may want to use the original
parameter reference later in that method code, and may not notice that it has already been changed.
As a conclusion, a method should not reassign its Object parameters.
Consider using copies of Object parameters and those can be reassigned without being prone to confusions or ambiguities.
Eventually, for the copies, use suggestive names such as copyOfFoo or tempFoo.
