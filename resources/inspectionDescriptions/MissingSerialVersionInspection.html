Serializable classes should provide a serialVersionUID field explicitly.
If a class implements Serializable and does not declare serialVersionUID explicitly,
it is automatically generated depending on the class details such as fields and methods.
This fact is ok, but the version number is automatically generated and changes every time the class changes.
Therefore, two different versions of the same class are automatically rendered incompatible without any developer action.
Well, this may cause InvalidClassException. More details: https://confluence.devfactory.com/x/DiakEw
