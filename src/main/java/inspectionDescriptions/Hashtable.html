Hashtable is specific to legacy code and has the main problem in that it's synchronized.
This means that it will be slow in any circumstance where it is shared between threads.
ConcurrentHashMap is a better choice in that situation.
On the other hand, on a single thread, the un-synchronized HashMap is a better choice.
In this case, consider using {{ConcurrentHashMap/HashMap}}.
